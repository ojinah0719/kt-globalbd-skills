# Project Lifecycle — 11단계 I/P/O · 역할 · 산출물 · 법률/기밀 요소

본 문서는 PMO가 **모든 참여자(KT·MyTV·VNPT)의 컨센서스**를 위해 사용하는 프로젝트 전체 흐름 정의입니다.
각 단계는 **Input → Process → Output** 으로 기술하며, **Lead(주관)·산출물·Exit 결론(다음 단계 진입 조건)·법률/기밀 요소**를 명시합니다.
5단계 Phase(`pmo-framework.md`)와의 매핑 및 회의 유형(`meeting-governance.md`)을 함께 제공합니다.

## 2. 단계별 I/P/O 카드

### 단계 1 — 솔루션 소개 ✅
- **I:** 시장·고객 니즈, KT 솔루션 카탈로그 / **P:** 솔루션 소개 PT / **O:** 소개자료, 초기 관심영역
- Lead KT · Exit: 후속 논의 합의 · 법률: 공개자료 수준

### 단계 2 — 제안 ✅
- **I:** 8 WS 니즈, 솔루션 적합성 / **P:** 제안서 작성·제출 / **O:** Business Cooperation Proposal
- Lead KT · Exit: 제안 검토·RFI 발행 · 법률: 제안서 Confidential 표기

### 단계 3 — 컨설팅/Deep-dive ▶ (현재)
- **I:** 제안서, RFI 7항목 / **P:** 온라인 미팅 — 제안 상세 설명, 정합/BM/인증 쟁점 deep-dive / **O:** 쟁점 정렬표, KT 입장 1pager, 회의록, RFI 대응 방향
- Lead 공동 · **Exit 결론(필수):** ① RFI 정식 회신 방향 합의 ② 정합·BM·인증 3대 쟁점 정렬 ③ 요구사항 워크숍·MOU 로드맵 합의 · **법률: NDA 체결(Deep-dive 진입)**

### 단계 4 — 요구사항 분석
- **I:** RFI, 미팅 결과, VNPT 시스템 정보(As-Is) / **P:** 요구사항 수집·분석, As-Is/Gap 분석, 정합 가능성 분석 / **O:** 요구사항 분석서, RTM 확정, As-Is/Gap 분석서
- Lead 공동(KT 기술) · Exit: 요구사항 baseline 합의 · 법률: NDA 하 시스템·기술정보 공유(As-Is = 기밀)

### 단계 5 — 요구사항 정의·과제범위 정의
- **I:** 요구사항 분석서, RTM / **P:** 요구사항 확정(SRS), 과제범위(SOW Scope) 정의, In/Out·변경통제 기준 / **O:** 요구사항 정의서(SRS), 과제범위정의서(SOW)
- Lead 공동 · Exit: 범위·요구사항 동결(Baseline) → 계약 진입 · 법률: SOW = 계약 부속(협상 기밀), MOU→본계약

### 단계 6 — 상세 서비스 시나리오 정의
- **I:** 요구사항 정의서 / **P:** 서비스 시나리오·UX 플로우·Use Case 상세화 / **O:** 서비스 시나리오 정의서, UX 플로우, Use Case 명세
- Lead 공동(MyTV 제품) · Exit: 시나리오 확정 → 설계 입력 · 법률: 제품 로드맵·시나리오 기밀

### 단계 7 — 시스템 설계
- **I:** 요구사항 정의서, 서비스 시나리오 / **P:** 아키텍처·인터페이스(IDD)·데이터·보안(데이터주권) 설계 / **O:** 시스템 설계서, IDD, To-Be 아키텍처, 데이터주권 설계서
- Lead KT(MyTV 레거시 협조) · **Exit: 설계검토(DR) 승인 → 개발 착수 (P1 종료 게이트)** · 법률: 아키텍처·I/F 양사 기술 기밀(NDA 핵심)

### 단계 8 — 개발 (8a 현지화 / 8b 정합·통합)
- **I:** 시스템 설계서, IDD / **P:** 현지화 개발(8a), 정합/통합 개발(8b), 형상관리 / **O:** 빌드(APK), 통합 모듈, 개발 산출물
- Lead KT · Exit: 개발 완료·단위테스트 통과 · 법률: 소스/IP 귀속(License/IP 매트릭스), NDA

### 단계 9 — 테스트 (단위→통합→E2E→UAT)
- **I:** 개발 산출물, 테스트 시나리오 / **P:** 단위/통합/E2E 테스트 → UAT / **O:** 테스트 결과서, 결함 대장, UAT 증빙
- Lead KT(개발 테스트)·공동(UAT) · Exit: 테스트/UAT 합격 기준 달성 · 법률: 테스트 데이터(개인정보 → PDPL 2025·Decree 356)

### 단계 10 — 검증 (인증·SLA·컴플라이언스)
- **I:** 테스트 결과, 인증 요건 / **P:** MIC ICT·Google Cert 검증, SLA 검증, 데이터주권 검증 / **O:** 인증서, 검증 보고서, 컴플라이언스 확인서
- Lead 공동 · **Exit: 인증 취득·SLA 충족 → 시범 (P2 종료 게이트)** · 법률: 인증·규제(현지 하드필터)

### 단계 11 — 시범/상용
- **I:** 인증·검증 완료 / **P:** 파일럿 → 상용 전환 → 운영 이관 → 확산 / **O:** 파일럿 결과, 상용 Go-Live, 운영 이관서
- Lead MyTV(실행)·KT(지원) · Exit: 상용 안정화·운영 이관 → 차기 WS · 법률: 상용 계약·SLA 운영·Rev Share 정산

## 3. 단계별 법률·기밀(NDA) 요소 진행 (Legal Progression)

사업이 진행될수록 공유 정보의 민감도와 계약 구속력이 커집니다. PMO는 단계마다 **무엇이 기밀이고 어떤 계약 장치가 필요한지**를 사전 관리합니다(요소 ⑫ 계약·IP 연계).

| 구간 | 단계 | 법률·기밀 장치 | 핵심 관리 포인트 |
|---|---|---|---|
| 탐색 | 1–2 | 일반 비밀유지, 제안서 Confidential | 공개 가능 범위 관리 |
| **Deep-dive 진입** | **3** | **NDA(상호 비밀유지계약)** | **사업·기술 비밀 공유 전 체결 — 본 미팅의 선행조건** |
| 분석·정의 | 4–6 | NDA 적용 + As-Is/제품 로드맵 기밀, (5)SOW=계약 부속 | VNPT 시스템·제품 정보 보호, MOU 착수 |
| 설계·개발 | 7–8 | NDA 핵심 적용 + **IP/License 귀속 · FCT(세금)·기술이전 등록(MOST)·언어(베-영 우선본)**, 소스·아키텍처 기밀 | 본계약 체결, IP·라이선스·세금·언어 명문화 |
| 검증 | 9–10 | **개인정보·데이터주권(PDPL 2025·Decree 356)**, 인증 기밀 | 데이터 거주·처리 컴플라이언스 |
| 상용 | 11 | 상용 계약·SLA·Rev Share 정산·페널티 | 운영 책임·정산·하자보수 |

**NDA 체크포인트(단계 3, 현재):**
- 형태: 상호(Mutual) NDA · 당사자: KT ↔ VNPT/MyTV · 준거법: 협의(베트남/한국)
- 범위: 사업 모델·원가·기술 사양·아키텍처·시스템 정보·고객 데이터
- 기간·예외·반환/파기 조항 · 본 미팅 일정 전 또는 미팅 시 체결 권장
- 심층 검토는 `L5-bd-contract-review`(준거법 베트남 지원) 위임.

**계약 체결 포인트 (베트남 — 본계약/라이선스 단계):**
- **형식·언어:** 일반계약은 언어 자유(영-베 이중언어 + **우선본 조항** 표준). 단 **기술이전·IT는 베트남어** 요구 가능 → 기술이전 시 **MOST 등록** + 베트남어 계약, 법원·공증 제출 시 베트남어 번역. 실행은 **법적대표자 서명 + 회사 직인(con dấu)**.
- **세금(FCT):** 비거주자 KT 지급액에 VAT+CIT 원천징수(**VNPT가 원천징수·납부**). 분류(기기=goods / 라이선스=royalty CIT 10% / 서비스)·**gross-up**·**한–베 조세조약** 적용을 계약에 명문화 — 마진 직결.
- **분쟁해결:** **VIAC**(베트남, 집행 신속) 또는 **SIAC/ICC**(중립, 단 외국판정은 베트남 법원 승인 필요). 뉴욕협약 가입, **국영기업 주권면제 배제**(중재 합의 시). 중재조항은 **서면·인정된 기관 명시**, 외국요소로 **영어 중재** 가능.
- **데이터:** PDPL 2025·Decree 356 — VNPT Cloud 거주 + DPIA/국외이전영향평가(CTIA) + 컨트롤러/프로세서 계약 + DPO.

## 4. 컨센서스 원칙 (참여자 공통 이해)
1. 모든 단계는 **I/P/O가 명확**해야 하며, Output은 다음 단계의 Input이 된다(추적 가능).
2. 단계 전환은 **Exit 결론(합의·승인)** 을 충족해야 한다(Stage-Gate).
3. 모든 산출물은 **Deliverable Registry**(⑥/.xlsx)에 ID·버전·상태·책임으로 등록된다.
4. 회의의 결론·의사결정은 **Decision Log**에 누적되어 최종 **계약서로 추적·반영**된다(`meeting-governance.md`).
