# Logistics, Certification, After-Sales & SLA — 물류·인증·AS·품질 통제 (RFI #6)

> 목적: 단말(8K STB·사운드바)의 **수입·통관·인증·창고·AS·정량 SLA**를 통제하고, VNPT 대상 **과다 확약**을 차단한다. **핵심 원칙: 프로세스·프레임워크는 확약하되, 정량 SLA(AFR·Epidemic 임계)·보증 조건·수입/수리 주체는 OEM 백투백 + 법무 확인 전까지 placeholder.** 세무·통관은 통관사, 계약·SLA 조항은 `L5-bd-contract-review`, 베트남 규제는 `vietnam-legal-checklist.md`, 정산·백투백은 `settlement-integrity.md`.

## 1. 수입·통관 (Customs & Importation)
- **IOR(수입자):** KT(한국법인)는 베트남 IOR 불가 → **VNPT 또는 VNPT 지정 현지 디스트리뷰터를 IOR**(현지법인·수입라이선스·매입VAT 환급 활용). KT는 합의 Incoterms로 인도.
- **Incoterms:** `[CIF/DAP]` Hai Phong/Hanoi 등 — 운임·보험·위험이전 분기점 명시.
- **분류·세금:** HS코드 `[8528 STB / 8518 사운드바]`, 수입관세 + **수입VAT 10%**. **한–베 FTA 특혜관세**(원산지증명 **CO Form VK/AK**)로 관세 경감 가능. 선적 전 통관사와 분류 합의(지연 방지).
- **서류:** 상업송장·packing list·CO·B/L·적합성 인증서 일체.
- **FCT 연계:** 단말=goods (`settlement-integrity.md` §3).

## 2. 창고·보관 (Warehousing)
- VNPT/디스트리뷰터가 배포 지역 인근 창고 운영. KT는 **보관 사양**(온습도·ESD/정전기 취급), FIFO, 시리얼 단위 추적 제공.
- **안전재고 = `[ ]`주 수요 + AFR 기반 예비부품**. 보유비용·보험은 Incoterms·보관계약에 따름.

## 3. 규제 테스트·MIC 인증 (Certification) ★게이트
- **MIC 타입승인(ICT/RF)은 상용 배포 전 필수 게이트.** 미취득 시 진행 불가.
- KT는 기술자료·기존 국제 시험성적서(CE/FCC 등)·**시험 샘플** 제공, 현지 시험 지원. 현지 신청주체가 필요하면 **VNPT/현지 파트너가 인증 보유자, KT는 기술 스폰서**.
- **병행 트랙: MIC ICT + Google Certification.** 샘플 수량·신청주체·일정을 조기 확정(파일럿 2027.1~3을 게이트).

## 4. After-Sales — 3-tier 모델
| Tier | 주체 | 범위 |
|---|---|---|
| **L1** | VNPT 콜센터/현장 | 1차 진단·DOA 교환·고객 응대 |
| **L2** | 현지 인증 서비스센터 | 보드 단위 수리·부품 교체·리퍼 (KT가 교육·매뉴얼·진단툴·예비부품 목록 제공) |
| **L3** | KT/OEM depot·엔지니어링 | 복합·에스컬레이션 결함·근본원인 분석 |
- **워크플로우:** 결함 접수 → RMA 발행 → L1 진단(DOA → `[ ]`일 내 선교환) → L2 수리/교체 → 불량품 역물류 → 예비부품 보충.
- **보증:** 제안 `[12~24]`개월(개통 기준). 보증 범위·제외는 보증약관. **KT→VNPT 보증·AS = OEM→KT 보증과 동일(백투백).**

## 5. 정량 SLA — AFR
- **정의·산식:** AFR = (필드 반품 ÷ 가동 설치대수), 정의된 기간 연환산. **SKU별 계약 목표**.
- **목표(placeholder):** 8K STB **`[≤ X%]`**, 사운드바 **`[≤ Y%]`**(단순 HW로 통상 더 낮음). OEM 신뢰성·필드 데이터로 확정, SLA에 명기. **월간 AFR 리포팅 + 공동 품질 리뷰.**
- 프리미엄 STB 업계 참고치는 낮은 한 자릿수 수준(확약 아님, OEM 확인 필요).

## 6. Epidemic (Mass) Failure 정책
- **정의:** **단일 공통 근본원인**에 의한 필드 결함이 합의 임계를 초과.
- **임계(제안):** 모델/배치별, rolling `[ ]`개월 내 누적 결함률이 (a) 배포대수의 `[N]%` 또는 (b) 합의 AFR의 `[k]×` 중 **큰 값** 초과.
- **메커니즘:** 트리거 → `[ ]`일 내 공동 근본원인분석(8D) → 원인별 구제: SW/펌웨어 → **OTA 수정** / 국소 부품 결함 → **대상 배치 교체** / 시스템·안전 HW 결함 → **리콜**.
- **비용 배분:** 교체·배상·리콜 비용은 **OEM 백투백 책임**에 따름. 구제·상한(cap)·기한은 SLA/계약에 정의.

## 7. Red Flags (정산·약속 오류 방지)
| 위험 | 통제 |
|---|---|
| AFR 과다 확약 | OEM 데이터 기반·placeholder·산식 합의 후 확정 |
| Epidemic 임계 모호 | N%/k×AFR 정량 정의·8D·비용배분 명시 |
| IOR·통관 주체 불명 | VNPT/디스트리뷰터 IOR·Incoterms·HS·CO 사전 확정 |
| MIC 인증 지연 | 병행 트랙·샘플·신청주체·일정 조기 고정·상용 전 게이트 |
| 로컬 AS 공백 | 3-tier·교육·매뉴얼·예비부품·역물류 책임 명시 |
| SLA > OEM 보증 | 모든 SLA/AS/보증·Epidemic 비용 **OEM 계약 flow-down** |

## 판정
- 1~6 프레임워크는 확약 가능. **정량 수치(AFR·Epidemic 임계·보증·통관)는 OEM 백투백·법무(L5) 확인 전까지 placeholder.** 미확정 항목은 공동 워킹세션에서 합의 후 SLA·공급/보증계약에 확정.
