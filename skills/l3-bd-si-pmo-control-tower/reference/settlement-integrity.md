# Settlement Integrity — 협력사 → KT → VNPT 정산 무결성 (점검·통제)

> 목적: 다단계 정산에서 **견적·세금·계약·송장·지급 간 불일치(정산 오류)** 를 사전 차단한다. 세율·세무 처리는 **세무사 확인 전제**, 심층 계약은 `L5-bd-contract-review`, 베트남 세무·데이터 상세는 `vietnam-legal-checklist.md`. 모든 금액은 **계획 추정치 + 가정 명시**(정식 견적 별도). 운영은 모드 C + 문서관리 SSOT 연동.

## 1. 정산 체인 맵
- **업스트림 (협력사 → KT):** KT가 솔루션 파트별로 구매/라이선스(STB HW·사운드바·AI Agent·TMS·OTT 등). 통화 KRW/USD, 한국 세제.
- **다운스트림 (KT → VNPT):** KT가 통합 솔루션 공급. 통화 USD/VND, 베트남 **FCT**.
- **원칙 — 백투백(Back-to-Back):** KT가 VNPT에 지는 의무·조건이 협력사 계약으로 **흘러내려야(flow-down)** KT 중간 리스크가 최소화된다.
- **유형 구분:** 기기=goods / SW=license(royalty) / 통합·구축=service → **세금·정산 방식이 유형별로 다름**.

## 2. 견적서(Quotation) 표준 & 단일 원가 기준(Single Source of Cost Truth)
- **라인아이템 구조:** 솔루션 파트 | 항목(HW/License/SI/현지화/인증/물류/AS) | 단가 | 수량(MOQ) | 통화 | 합계 | **유형(goods/royalty/service)** | **세금 라인** | 비고
- **원가 빌드업 3단 일치:** 협력사 원가 → KT 마진 → VNPT 가격. 각 라인은 SOW·범위와 **1:1 매핑(scope-line mapping)**.
- **버전·단일 기준:** 견적은 버전 관리(문서관리 Register 연동), **확정(binding) vs 추정(estimate)** 명시. 모든 정산은 **동일 원가 기준 1개**에서 파생.
- **통화·환율 기준일·유효기간** 명시.

## 3. 세금 처리 관계(Tax Treatment) — 체인 캐스케이드
> 세율·적용은 세무사 확인. 베트남 측 상세 = `vietnam-legal-checklist.md` §1(FCT).
- **협력사 → KT (한국):** 국내 VAT(매입세액), 해외 협력사면 한국 원천징수, **특수관계 시 이전가격(transfer pricing)** 문서화.
- **KT → VNPT (베트남):** **FCT(VAT+CIT)** — VNPT 원천징수. 파트별 분류(기기=goods 저율 / SW=royalty CIT 10% / 서비스) → **번들을 세금 목적상 언번들(unbundle)**.
- **이중과세·gross-up 캐스케이드:** 각 링크에서 **누가 어떤 세금을 부담**하는지(gross-up 여부) 명시 → 세금이 마진을 잠식하지 않도록. **한–베 조세조약(DTA)** 으로 로열티·CIT 경감 적용.
- **분류 일관성:** 동일 솔루션 파트의 세금 분류가 **협력사↔KT↔VNPT 계약에서 일치**해야 한다(불일치 시 추징·정산 오류).
- **세금 라인 분리:** 세금을 견적·계약·송장에 **명시적 라인**으로 분리 표기.

## 4. 계약서(Contract) — 백투백 Flow-Down
KT–VNPT 본계약과 협력사 계약에 **동일하게** 반영(불일치 = KT 중간 리스크):
- 범위·산출물·검수 기준 · **지급 마일스톤·검수연동(acceptance-linked)** · 통화·환리스크 부담 · **MG·Rev Share의 협력사 배분** · 하자·페널티·SLA **pass-through** · IP/License 체인 · 계약기간·해지 정합 · **세금 부담·gross-up·분류**.
- 변경(Change) 시 가격 영향 = ①변경통제 → ③·④ Baseline 갱신 → **협력사·VNPT 양방향** 반영.

## 5. 정산 오류 방지 — N-way 대사 통제(Reconciliation)
- **다중 대사(N-way match):** 견적 ↔ 계약/PO ↔ 납품·검수 ↔ **송장(Invoice)** ↔ **지급(Payment)** — 수량·단가·통화·세금 라인 일치 확인.
- **검수 연동 지급:** 마일스톤·검수 합격 시에만 지급(미검수 지급 금지).
- **통화·환 정책:** 계약 통화·환율 기준일·헤지/조정 조항 통일. 협력사(KRW/USD)↔VNPT(USD/VND) **환율 갭** 관리.
- **변경 통제:** 범위·수량·단가 변경은 정산 전 승인·Baseline 갱신(임의 정산 금지).
- **감사 추적:** 견적·계약·송장·지급·세금계산서 보관·링크(문서관리 SSOT).
- **단일 책임:** **PMO+재무**가 정산 대사 소유(R), 정기(월/마일스톤) 대사.

## 6. 정산 RACI & Red Flags
| 오류 모드 | 통제 |
|---|---|
| 견적 ↔ SOW 라인 불일치 | scope-line 1:1 매핑·버전 단일 기준 |
| 세금 분류 오류(goods vs royalty) | 파트별 분류표·체인 일관성·세무사 확인 |
| 환율 드리프트 | 통화·기준일·조정 조항 통일 |
| 이중 계상·누락 | N-way 대사·단일 원가 기준 |
| 미검수/미승인 변경 지급 | 검수연동 지급·변경통제 |
| MG·Rev Share 오산 | 산식 문서화·협력사 배분 명시 |

- **R** = PMO+재무 / **A** = 사업총괄 / **C** = 세무·법무(`L5-bd-contract-review`) / **I** = 협력사·VNPT.

## 판정
- 1~5 통제 모두 ✅ → 정산 진행. 미흡 항목은 ③(원가/BM)·⑫(계약)·⑧(조달)에서 보완하고, 세무·법무는 자문 의뢰 후 **재점검**한다.
